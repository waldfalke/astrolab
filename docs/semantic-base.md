## Модель: семантические резервуары и роли
Значение — не строка, а **веер** (вектор смысла). Делинеация — **некоммутативная композиция**
этих векторов («перемножение матриц», AB ≠ BA). Композиция определена, потому что у каждого
элемента — своя **роль**:

| Роль | Элемент | Что несёт |
|---|---|---|
| **Деятель** (кто) | планета | мотив/функция, глагол |
| **Резервуар / качество** (как окрашен) | знак | стихия/крест/темп, стиль |
| **Арена** (где, область жизни) | дом | поле проявления |
| **Направленная связь** (как сцеплены) | аспект | модификатор между деятелями |
| (сила деятеля) | достоинство | усиление/ослабление |
| (норма/вне нормы) | склонение/OOB | в рамках / нерегулируемо |

**Валентность не предрешена.** Каждый элемент даёт диапазон (связь↔вязкость, воля↔разрушение).
Какой полюс — решает **наблюдение и корроборация**, не текст заранее (продолжение двухосной модели
и правила «не сочинять вперёд»).

## Операторы композиции (направленные — порядок и роль фиксированы)
Читатель собирает положение, **применяя операторы**, а значения берёт из терсовых таблиц ниже.

1. **планета ⊗ знак** → деятель, окрашенный резервуаром. (Венера⊗Телец = «ценить» в
   телесно-устойчивом стиле.)
2. **планета ⊗ дом** → деятель, действующий в арене.
3. **управитель(A) → B** (диспозиция) → тема дома/знака A **проведена через** B. *Направлено:*
   «управитель A в B» ≠ «управитель B в A».
4. **аспект (быстрая → медленная)** → направленная связь-модификатор двух деятелей.
5. **N-й от X** (производный дом) → топик N **относительно** поля X. *Направлено:* 2↔8 (см. ниже).

**Результат любой композиции — веер**, не один смысл. Валентность открыта до наблюдения.

## Некоммутативность (важнейшее)
- **Смысловая.** «связь 2 и 8» ≠ «связь 8 и 2». Доход от общего (2-й **от** 8-го) ≠ кризис/трата
  своего (8-й **от** 2-го) — **даже когда оба указывают в один дом** (9-й): клетка одна, путь
  разный. Это же — направление аспекта (быстрая→медленная), диспозиции (A правит знаком B),
  деривации (N-й *от* X).
- **Процедурная.** Порядок чтения инструментов меняет результат: соляр→натал ≠ натал→соляр (что
  прочитано первым — задаёт рамку). См. «Типовые ритуалы».

## Каталог антипаттернов (то, что наработано — собрано)
1. **Детерминированный коллапс** — «положение → одно фиксированное событие» («Марс в 7 = развод»). Положение даёт веер, не приговор.
2. **Одновалентность** — взять один полюс (8-й = только смерть), игнорируя веер.
3. **Предрешённая валентность** — решить «хорошо/плохо» до наблюдения. Полюс выбирает факт, не текст.
4. **Перечисление произведения** — писать клетки «планета×знак×дом» вместо операторов. Если пишешь «Марс в Раке значит…, Марс во Льве значит…» — стоп, это произведение.
5. **Симметризация направленного** — читать 2↔8, диспозицию, аспект, деривацию как коммутативные. Они направлены.
6. **Путаница уровней** — натал «показывает» текущую ситуацию/событие. Натал = стоячая структура; «когда/сейчас» — тайминг (транзит/прогрессия/дирекция).
7. **Постдикция-подгонка** — подгонять символ под уже известную биографию. Фиксируй структуру, наблюдай окно.
8. **Дешёвое подтверждение** — считать слабый/быстрый сигнал за полноценное указание. Правило трёх для слабых/косвенных; быстрый триггер ≠ медленное подтверждение.
9. **Орб без знака** — читать separating (прошлое) как applying (будущее). Решает знак, не модуль орба.
10. **Пустое определительное** — определяющее по построению (Солнце соляра ☌ натальное Солнце) сигнала не несёт.
11. **Геометрия вместо таблицы** — выводить экзальтацию из фазовой геометрии. Экзальтация/падение — таблица; обитель/изгнание — геометрия.
12. **Преждевременный синтез** — синтезировать до полного обхода всех факторов (см. «Полнота обхода»).
13. **Импорт из чужой системы** — мешать дома разных систем / уровни / школы в одном шаге.
14. **Черипик (выборочная подсветка)** — поднять удобные факторы/полюса, умолчав о неудобных. Отлично от #2/#3 (там — валентность одного фактора): здесь — **селекция по всему чтению**. Лечится полнотой обхода: каждый фактор обязан получить явную диспозицию.

## Полнота обхода (coverage) — карта рассмотрена целиком

**Определение.** Карта рассмотрена в полноте, когда **каждый фактор, присутствующий в данных,
получил явную диспозицию** — и ни один не выпал молча. Диспозиций три:
1. **несущий** — втянут в трактовку;
2. **тихий** — присутствует, но малозначим: явно помечен и отложен (**не** выброшен);
3. **вне охвата** — исключён по решению о scope (минорные аспекты, жребии, звёзды — пока сюда).

Полнота — про **обход**, не про объём прозы: **обходим всё, пишем несущее, тихое помечаем тихим**.
Множество факторов **ограничено** (кардинальность данных, не произведение — иначе это антипаттерн #4):
объекты + их состояния (знак · дом · достоинство · секта · ретро · склонение/OOB), аспекты (**из
списка, не из памяти**), конфигурации, производное (управитель карты · светило · стоки-диспозиторы ·
балансы), и по каждой технике — тот же обход заново + кросс-связи с наталом.

**Salience динамична — её ставит корроборация.** Несущий/тихий — не свойство фактора в одиночку, а
функция схождения по инструментам. Диспозиция — **не write-once**: каждый следующий инструмент может
её **поднять / опустить / переоткрыть**. Поэтому реестр **живой**, и техники читают и пишут в **один
общий реестр** поверх всего массива знаний карты — это **проходы**, а не изолированные истории
(изоляция = «не дать инструментам переоценить друг друга»; так был потерян стеллиум соляра).

**Два схлопывания — законно только одно.**
- **Какая арена** (какой дом/тема горит) — схлопывается законно, через *подтверждение отсутствия*:
  тихие зоны исключаются. Это и есть нормальное сужение прогноза.
- **Какой полюс внутри живой арены** — в прогнозе **не** схлопывается сам собой: оба полюса живы,
  пока не наблюдены. Свернуть полюс/тему можно только по **основанию** — *корроборация между
  инструментами* или *наблюдение* (биография), — но не «красиво». Свёртка без основания = **черипик**
  (#14). Натал иногда сверяется с биографией, будущий год — нет, поэтому прогноз **валентно-открытее**
  натала, а не наоборот.

**Готово = реестр стабилизировался.** Не «все галочки проставлены один раз» и не «не осталось что
дорасследовать» (на живой карте всегда осталось). Готово, когда **новый проход не двигает ранги
значимо** — fixpoint **по бюджету**.

**Артефакт — реестр обхода в виде keyed-контракта из трёх файлов** (рецепт
`artifacts/mcp-recipes/build_coverage_ledger.ps1`). Общий ключ — `factor_id`; он держит стык
*скрипта* (перечисляет и считает) и *модели* (судит), не давая одному делать работу другого:

1. **`coverage_factors.csv`** — машинный. Каждый фактор из данных + устойчивый `factor_id`.
   Перегенерируется свободно (факторы из данных, не из памяти).
2. **`coverage_dispositions.csv`** — семантический, его **владелец — модель**. По ключу: `salience`
   (несущий/тихий/вне охвата) · `valence_resolved` · `basis` (список `factor_id`) · `note`. Скрипт
   **никогда не переписывает** существующие строки — только **дописывает** новые ключи в хвост
   (правки переживают регенерацию по построению, а не «бережным слиянием»).
3. **`coverage_report.md`** — проверяльщик (read-only): JOIN первых двух по ключу + структурные
   проверки + полная сводная таблица + тэлли.

**Ключ направлен — концы аспекта НЕ сортируются.** Кросс-рамочные аспекты несут направление
(соляр-Сатурн→натал-Луна ≠ натал-Сатурн→соляр-Луна); сортировка симметризовала бы направленное
(антипаттерн #5). Устойчивость даёт детерминированный обход одного и того же файла, не сортировка.

**Три структурные проверки (механизируемая часть приёмки):**
- **полнота** — у каждого `factor_id` непустой `salience` (иначе дыра / «лысый соляр»);
- **целостность оснований** — каждый `factor_id`, указанный в `basis`, существует в данных. Это
  **против выдуманных оснований** (ловит фантомную корроборацию — как занос аспектов из чужой
  карты), а **не** против черипика: избирательное замалчивание реального фактора (тихо сделать
  «тихим», чтобы обойти) проверки не ловят — это остаётся **бременем модели**, не скрипта;
- **полюс ⇒ основание** — `valence_resolved=yes` обязывает `basis` быть непустым (нет «красивой»
  свёртки полюса). Поэтому *вес* (salience) и *выбор полюса* (valence_resolved) разведены в схеме.

Тихое и исключённое **сохраняются видимо** — отсутствие тогда аудируемо, а не выглядит как «всё
охвачено». **Пустая диспозиция = молчаливая дыра**, чинить до сдачи.

## Типовые ритуалы (порядок чтения — некоммутативен)
- **«Понять человека»:** только натал (структура), без таймингов.
- **«Год вперёд»:** натал (структура-рамка) → профекция (хозяин года) → соляр (тема года) →
  solar arc (многолетняя рамка) → прогрессия (таймер) → транзиты (тонкий тайминг). Каждый
  следующий читается **внутри рамки** предыдущего; рамочные не датируют неделю, быстрые — датируют.
- Правило: что прочитано первым — задаёт рамку для остального. соляр→натал ≠ натал→соляр.
- **Обход некоммутативен И возвратен.** Порядок задаёт рамку (выше), но проходы **повторяются**:
  увидел в дирекциях/транзитах тему — вернулся и пере-разметил **общий реестр** (см. «Полнота
  обхода»). Линейного «один раз сверху вниз» не бывает — крутим до сходимости рангов.

## Проверка приёмки: воспроизводимо ли чтение из ОДНОГО дока
Берём уже сделанный вывод и собираем его **только из операторов + таблиц** этого дока. Где не
вышло — там дыра, молча заполненная из головы.

**Пример — «Венера в Тельце в 8-м, корень карты»:**
- ♀ (деятель: сближать/ценить; веер: связь↔вязкость) **⊗ Телец** (резервуар: телесное/устойчивое/
  медленное) → «ценить» в вещественно-устойчивом стиле = **ценность телесная, традиционная, медленная**.
- **⊗ 8-й дом** (арена: общий/чужой ресурс/раздел/глубина) → эта ценность проявляется в области
  **общего ресурса и его раздела**.
- **корень = диспозитор-сток** (оператор 3: большинство цепочек проведены к Венере) → этот деятель —
  **гравитационный центр**, вся карта разворачивается к нему.
- Композиция (веер, валентность открыта): **«телесно-традиционная ценность, проживаемая через
  общий ресурс / раздел / глубину»** — ровно прежний вывод. ✓ Воспроизведено из дока.
- Валентность (раздел vs обладание vs интимность) — **не предрешена**, выбирается наблюдением.

Критерий: новый вывод считается готовым, только если так же собирается из дока. Не собирается →
доработать таблицу/оператор, а не дописать смысл «из головы».

**Второй критерий приёмки — полнота.** Чтение готово, только если **реестр обхода закрыт**: у каждого
фактора непустая диспозиция, тихое помечено тихим (не выброшено), ранги стабилизировались. Пустая
клетка = молчаливый пропуск, чинить до сдачи. (См. «Полнота обхода».) Два критерия независимы:
воспроизводимость ловит «дописал из головы», полнота ловит «пропустил/черипикнул».

## Вынесено за «базовую» (явный потолок)
- Минорные аспекты; значения жребиев и неподвижных звёзд (в роадмапе отдельно).
- Исчерпывающие таблицы по каждой комбинации (это и есть «перечисление произведения» — запрещено).
- Формальная векторизация значений (структурные данные для машины) — дверь оставлена открытой
  (ключи атомарны и единообразны), но в «базовую» не входит.
