## Модель: семантические резервуары и роли
Значение — не строка, а **веер** (вектор смысла). Делинеация — **некоммутативная композиция**
этих векторов («перемножение матриц», AB ≠ BA). Композиция определена, потому что у каждого
элемента — своя **роль**:

| Роль | Элемент | Что несёт |
|---|---|---|
| **Деятель** (кто) | планета | мотив/функция, глагол |
| **Резервуар / качество** (как окрашен) | знак | стихия/крест/темп, стиль |
| **Арена** (где, область жизни) | дом | поле проявления |
| **Направленная связь** (как сцеплены) | аспект | модификатор между деятелями |
| (сила деятеля) | достоинство | усиление/ослабление |
| (норма/вне нормы) | склонение/OOB | в рамках / нерегулируемо |

**Валентность не предрешена.** Каждый элемент даёт диапазон (связь↔вязкость, воля↔разрушение).
Какой полюс — решает **наблюдение и корроборация**, не текст заранее (продолжение двухосной модели
и правила «не сочинять вперёд»).

## Операторы композиции (направленные — порядок и роль фиксированы)
Читатель собирает положение, **применяя операторы**, а значения берёт из терсовых таблиц ниже.

1. **планета ⊗ знак** → деятель, окрашенный резервуаром. (Венера⊗Телец = «ценить» в
   телесно-устойчивом стиле.)
2. **планета ⊗ дом** → деятель, действующий в арене.
3. **управитель(A) → B** (диспозиция) → тема дома/знака A **проведена через** B. *Направлено:*
   «управитель A в B» ≠ «управитель B в A».
4. **аспект (быстрая → медленная)** → направленная связь-модификатор двух деятелей.
5. **N-й от X** (производный дом) → топик N **относительно** поля X. *Направлено:* 2↔8 (см. ниже).

**Результат любой композиции — веер**, не один смысл. Валентность открыта до наблюдения.

## Некоммутативность (важнейшее)
- **Смысловая.** «связь 2 и 8» ≠ «связь 8 и 2». Доход от общего (2-й **от** 8-го) ≠ кризис/трата
  своего (8-й **от** 2-го) — **даже когда оба указывают в один дом** (9-й): клетка одна, путь
  разный. Это же — направление аспекта (быстрая→медленная), диспозиции (A правит знаком B),
  деривации (N-й *от* X).
- **Процедурная.** Порядок чтения инструментов меняет результат: соляр→натал ≠ натал→соляр (что
  прочитано первым — задаёт рамку). См. «Типовые ритуалы».

## Каталог антипаттернов (то, что наработано — собрано)
1. **Детерминированный коллапс** — «положение → одно фиксированное событие» («Марс в 7 = развод»). Положение даёт веер, не приговор.
2. **Одновалентность** — взять один полюс (8-й = только смерть), игнорируя веер.
3. **Предрешённая валентность** — решить «хорошо/плохо» до наблюдения. Полюс выбирает факт, не текст.
4. **Перечисление произведения** — писать клетки «планета×знак×дом» вместо операторов. Если пишешь «Марс в Раке значит…, Марс во Льве значит…» — стоп, это произведение.
5. **Симметризация направленного** — читать 2↔8, диспозицию, аспект, деривацию как коммутативные. Они направлены.
6. **Путаница уровней** — натал «показывает» текущую ситуацию/событие. Натал = стоячая структура; «когда/сейчас» — тайминг (транзит/прогрессия/дирекция).
7. **Постдикция-подгонка** — подгонять символ под уже известную биографию. Фиксируй структуру, наблюдай окно.
8. **Дешёвое подтверждение** — считать слабый/быстрый сигнал за полноценное указание. Правило трёх для слабых/косвенных; быстрый триггер ≠ медленное подтверждение.
9. **Орб без знака** — читать separating (прошлое) как applying (будущее). Решает знак, не модуль орба.
10. **Пустое определительное** — определяющее по построению (Солнце соляра ☌ натальное Солнце) сигнала не несёт.
11. **Геометрия вместо таблицы** — выводить экзальтацию из фазовой геометрии. Экзальтация/падение — таблица; обитель/изгнание — геометрия.
12. **Преждевременный синтез** — синтезировать до полного обхода всех факторов (см. «Полнота обхода»).
13. **Импорт из чужой системы** — мешать дома разных систем / уровни / школы в одном шаге.
14. **Черипик (выборочная подсветка)** — поднять удобные факторы/полюса, умолчав о неудобных. Отлично от #2/#3 (там — валентность одного фактора): здесь — **селекция по всему чтению**. Лечится полнотой обхода: каждый фактор обязан получить явную диспозицию.

## Полнота обхода (coverage) — карта рассмотрена целиком

> **Где обойдённое выписывается:** `docs/chart-datasheet.md` — цифровой двойник. Обход даёт полноту;
> двойник делает её **видимой** холодным языком ПЕРЕД прозой (ритуал `compute → двойник → проза`) и
> ловит «тихое понижение» пропуском (чего нет на доске — не учтено). Этот файл — *как* читать; двойник —
> *как зафиксировать* прочитанное перед текстом.

**Определение.** Карта рассмотрена в полноте, когда **каждый фактор, присутствующий в данных,
получил явную диспозицию** — и ни один не выпал молча. Диспозиций три:
1. **несущий** — втянут в трактовку;
2. **тихий** — присутствует, но малозначим: явно помечен и отложен (**не** выброшен);
3. **вне охвата** — исключён по решению о scope (минорные аспекты, жребии, звёзды — пока сюда).

Полнота — про **обход**, не про объём прозы: **обходим всё, пишем несущее, тихое помечаем тихим**.
Множество факторов **ограничено** (кардинальность данных, не произведение — иначе это антипаттерн #4):
объекты + их состояния (знак · дом · достоинство · секта · ретро · склонение/OOB), аспекты (**из
списка, не из памяти**), конфигурации, производное (управитель карты · светило · стоки-диспозиторы ·
балансы), и по каждой технике — тот же обход заново + кросс-связи с наталом.

**Балон (#4) — грех ПРОЗЫ, а не размера описи.** «Не перечисляй произведение» = не вываливай клетки в
текст толкования; это **не** «держи реестр коротким». Выбросить реальные факторы из **обхода** ради
краткости — нарушение полноты (молчаливый черипик), а не защита от балона. Различающий вопрос между
«убрать» и «оставить»: фактор **топит прозу** или просто **лежит в описи, доступный для анализа**?
Если второе — оставить, пометив тихим.

**Непрерывные слои — два под-слоя, не фильтр.** Где факторов поток (транзиты за год — сотни реальных
проходов в орбе), разводим по скорости, а не вырезаем: **несущие** (медленные Юпитер…Плутон — темы
года, судятся как любой фактор) и **триггеры** (быстрые ☉☽☿♀♂ — датируют момент; авто-`тихий`,
поднимаются в несущие **только корроборацией**: попал на заряженную медленным точку / звено цепочки).
Так полнота цела (цепочки и атипичные сцепки видны — быстрый крючок на заряженной точке), а проза не
тонет. ⛔ Не «включать только те быстрые, что на заряженной точке» — это заранее решает, что атипично;
атипичное **всплывает из обхода**, не из догадки. **Полнота непрерывного слоя — относительна к scope**
(орб · тела · диапазон): scope объявляется в сводке прогона, тогда «полно» аудируемо.

**Salience динамична — её ставит корроборация.** Несущий/тихий — не свойство фактора в одиночку, а
функция схождения по инструментам. Диспозиция — **не write-once**: каждый следующий инструмент может
её **поднять / опустить / переоткрыть**. Поэтому реестр **живой**, и техники читают и пишут в **один
общий реестр** поверх всего массива знаний карты — это **проходы**, а не изолированные истории
(изоляция = «не дать инструментам переоценить друг друга»; так был потерян стеллиум соляра).

**Метод трактовки весомого транзит-события — веер → корроборация** (NKS realm `astrolab` #92).
Двухступенчатый, и снимает раздувание трактовок (в т.ч. производных домов, N-от-M) с двух концов:
1. **Веер «как может проиграться» + фильтр ВХОДА.** Линия проявления (включая производное значение
   дома) живёт в вее­ре **только при зацепке** — есть носитель (планета в доме / задействован
   управитель / транзит туда бьёт). Ни с чем не связано нигде → **отбраковать сразу**, в веер не пускать.
   Режет бесконечность на входе (есть-связь/нет — это свойство, вычислимо).
2. **Отбор.** Сильное (тесный орб, угол, очевидное схождение) — берём прямо. Неявное — несущее при
   **≥3 НЕЗАВИСИМЫХ подтверждениях при 0 опровержений**. Анти-черипик: подтверждение = независимый
   слой (транзит·дирекция·прогрессия·профекция·соляр), не одно тело трижды; «0 опровержений» = активный
   поиск против, не «не заметил». Связь работает дважды: бинарно на входе, счётно (независимые) на отборе.
   Масштаб — для весомых событий года/периода, НЕ для часов-внутри-дня (там утилитарный scoring).

**Два схлопывания — законно только одно.**
- **Какая арена** (какой дом/тема горит) — схлопывается законно, через *подтверждение отсутствия*:
  тихие зоны исключаются. Это и есть нормальное сужение прогноза.
- **Какой полюс внутри живой арены** — в прогнозе **не** схлопывается сам собой: оба полюса живы,
  пока не наблюдены. Свернуть полюс/тему можно только по **основанию** — *корроборация между
  инструментами* или *наблюдение* (биография), — но не «красиво». Свёртка без основания = **черипик**
  (#14). Натал иногда сверяется с биографией, будущий год — нет, поэтому прогноз **валентно-открытее**
  натала, а не наоборот.

**Готово = реестр стабилизировался.** Не «все галочки проставлены один раз» и не «не осталось что
дорасследовать» (на живой карте всегда осталось). Готово, когда **новый проход не двигает ранги
значимо** — fixpoint **по бюджету**.

**Артефакт — реестр обхода в виде keyed-контракта из трёх файлов** (рецепт
`artifacts/mcp-recipes/build_coverage_ledger.ps1`). Общий ключ — `factor_id`; он держит стык
*скрипта* (перечисляет и считает) и *модели* (судит), не давая одному делать работу другого:

1. **`coverage_factors.csv`** — машинный. Каждый фактор из данных + устойчивый `factor_id`.
   Перегенерируется свободно (факторы из данных, не из памяти).
2. **`coverage_dispositions.csv`** — семантический, его **владелец — модель**. По ключу: `salience`
   (несущий/тихий/вне охвата) · `valence_resolved` · `basis` (список `factor_id`) · `note`. Скрипт
   **никогда не переписывает** существующие строки — только **дописывает** новые ключи в хвост
   (правки переживают регенерацию по построению, а не «бережным слиянием»).
3. **`coverage_report.md`** — проверяльщик (read-only): JOIN первых двух по ключу + структурные
   проверки + полная сводная таблица + тэлли.

**Ключ направлен — концы аспекта НЕ сортируются.** Кросс-рамочные аспекты несут направление
(соляр-Сатурн→натал-Луна ≠ натал-Сатурн→соляр-Луна); сортировка симметризовала бы направленное
(антипаттерн #5). Устойчивость даёт детерминированный обход одного и того же файла, не сортировка.

**Три структурные проверки (механизируемая часть приёмки):**
- **полнота** — у каждого `factor_id` непустой `salience` (иначе дыра / «лысый соляр»);
- **целостность оснований** — каждый `factor_id`, указанный в `basis`, существует в данных. Это
  **против выдуманных оснований** (ловит фантомную корроборацию — как занос аспектов из чужой
  карты), а **не** против черипика: избирательное замалчивание реального фактора (тихо сделать
  «тихим», чтобы обойти) проверки не ловят — это остаётся **бременем модели**, не скрипта;
- **полюс ⇒ основание** — `valence_resolved=yes` обязывает `basis` быть непустым (нет «красивой»
  свёртки полюса). Поэтому *вес* (salience) и *выбор полюса* (valence_resolved) разведены в схеме.

Тихое и исключённое **сохраняются видимо** — отсутствие тогда аудируемо, а не выглядит как «всё
охвачено». **Пустая диспозиция = молчаливая дыра**, чинить до сдачи.

## Лог версий (видимость отбрасывания ветвей)

Реестр обхода аудитирует **факторы** (каждый → диспозиция, полюс⇒основание). Но коллапс веера к полюсу
происходит **внутри** фактора, и у языковой модели он систематически **молчалив и беглостный** — веер
схлопывается к привычному/клишированному полюсу, а реально-аффордируемые альтернативы часто **даже не
всплывают**. Это #2 (одновалентность) и #14 (черипик) на уровне **ветки**, и структурные проверки реестра
их не видят (в самом отчёте: «selective demotion stays the model's burden»).

**Правило.** Веер полюсов берётся **из таблиц-операторов**, не из того, что бегло всплыло (привязка к
воспроизводимости: нет полюса в доке — нет его и в чтении «из головы»). При свёртке к полюсу **каждый
реально-аффордируемый полюс логируется**: `taken` (взят) · `parked` (оставлен **живым**, не выбран —
законно) · `dropped` (свёрнут — **обязано основание**). Молчаливое исчезновение аффордируемого полюса
запрещено.

**Артефакт — `coverage_versions.csv`, четвёртый файл keyed-контракта** (тот же `factor_id`). Владелец —
**модель**, append-only: скрипт лишь заводит заголовок, строк не трогает (как диспозиции). Колонки:
`factor_id · pole · status(taken|parked|dropped) · basis(список factor_id / «наблюдение») · note`.

**Проверки (gate 3 — нет молчаливого схлопывания):** `valence_resolved=yes` ⇒ у фактора **есть** хотя бы
одна непустая версия (свернул полюс — покажи работу ветвей) · `dropped` ⇒ **основание непусто** ·
целостность основания версий · сироты. Скрипт **не перечисляет полюса** (веер — суждение модели), только
проверяет видимую часть.

**Честный потолок.** Лог ловит ветви, которые модель **рассмотрела**; полюс, который **не всплыл вовсе**,
не поймает ни одна проверка — это смягчается тем, что веер читается из оператор-таблиц, а не из памяти.
Поэтому gate 3 работает **в паре** с воспроизводимостью (gate 1), не вместо.

## Фазовый слой (авторская фаз-модель) — рабочий, не в подаче

Дополнительный слой состояния объекта поверх знак/дом/достоинство/секта/склонение. **Авторский,
копирайтный** (имя и проза-смыслы — в `docs/sources/`, gitignored). ⛔ **В клиентскую подачу не идёт**
(см. report-standards §4.1); живёт только в рабочем чтении и реестре обхода.

**Один фрактальный оператор.** `Φ(точка, опора) = ((idx_точки − idx_опоры) mod 12) + 1` — 12-цикл от
опорной точки. Вся машина — этот оператор по двум осям: **рамка** {домицил→Z, ASC→H} × **масштаб**
{знак 30°→Z,H · 2.5° внутри→z,h} + рекурсия диспозитора (D = Z управителя знака). Состояние —
`P⟨Z.z : H.h : D⟩`. Это «оператор, не произведение» (анти-#4): хранится оператор + алфавит 12
архетипов; таблица соответствий — лишь **самотест** (рецепт обязан её воспроизвести, иначе падает).

**Выпадает из оператора бесплатно** (признак, что принцип настоящий, а не подгонка): дубли ☿/♀ (два
домицила ⇒ два Z), покрытие высших (другой домицил, та же формула), неподвижная точка без домицила
(нет Z, но есть H), фаза цикла/возврата (у цикла свой старт-домицил).

**Два эпистемических тира (постоянные — потолок источников):**
- **Z, H — заземлены.** Z воспроизводит авторскую таблицу соответствий клетка-в-клетку (сверено,
  10 тел + дубли); H — тот же оператор на рамке ASC.
- **z, h, D — `anumita`.** Выведены тем же оператором **и** засвидетельствованы (семинары автора), но
  **источником не сверяемы** (книг по ним нет, софт утрачен). Это **не временно** — потолок. Помечать
  всегда; не давать «выведено» дрейфовать в «доказано».

**Аксиомы-входы (назвать честно):** схема управителей — **современная** (иначе у высших нет домицила и
цикл управителей фаз не замкнётся); это **конфликтует** с традиционной схемой профекций — держать
раздельно. Плюс: 12-архетипный алфавит, опорные точки (домицил/ASC), отсчёт вперёд (домицил = фаза 1;
направление проверяется правилом «фаза 7 = изгнание»).

**Как входит в обход.** Фаза — базовое свойство объекта (как достоинство): считается для **всех** тел
(`run_phase_vectors.ps1` → `phase_vectors.csv`, join по телу в реестр). Полнота = посчитано всем;
толкование — salience модели. Новый фаз-проход переранжирует реестр **корроборацией**, а не новой
ареной. **Горизонт (не сейчас):** виртуальные поля, истинные узлы планет, метод циклов — тот же
оператор на новых опорах; см. `methodology-roadmap.md`.

## Типовые ритуалы (порядок чтения — некоммутативен)
- **«Понять человека»:** только натал (структура), без таймингов.
- **«Год вперёд»:** натал (структура-рамка) → профекция (хозяин года) → соляр (тема года) →
  solar arc (многолетняя рамка) → прогрессия (таймер) → транзиты (тонкий тайминг). Каждый
  следующий читается **внутри рамки** предыдущего; рамочные не датируют неделю, быстрые — датируют.
- Правило: что прочитано первым — задаёт рамку для остального. соляр→натал ≠ натал→соляр.
- **Обход некоммутативен И возвратен.** Порядок задаёт рамку (выше), но проходы **повторяются**:
  увидел в дирекциях/транзитах тему — вернулся и пере-разметил **общий реестр** (см. «Полнота
  обхода»). Линейного «один раз сверху вниз» не бывает — крутим до сходимости рангов.

## Чтение по сферам (ось «область жизни» × ось «линза»)

Прогноз года собирается по **двум осям**: *сферы* (область жизни — дома, свёрнутые в узнаваемые) и
*линзы* (инструменты). По-сферное чтение = синтез вниз по столбцу «сфера» через все линзы.
**Соляр — рама сборки** (год ведёт соляр, синтез сидит в слое года; новой плумбинги не нужно).

**Роли линз — что каждая даёт сфере:**

| Линза | Вклад в сферу |
|---|---|
| натал | **база** — что область *есть* (стоячая структура, не «сейчас») |
| профекция | **прожектор года** — профектируемый дом **+ положение его управителя**; тянет 2–3 сферы (рифма года) |
| соляр | **раздача заряда** по сферам (дома/управители соляра) + **узлы** (соляр→натал жёсткие аспекты) |
| прогрессия | **медленный сдвиг** (смена знака) в немногих сферах |
| дирекция (солн. дуга) | **точечная перфекция** по сфере — датируемый узел |
| транзитные окна | **тайминг** — *когда* узел сферы активируется |

**Матрица разрежённая и взвешенная.** Линза говорит про сферу только там, где реально подсвечивает;
заполнять все клетки = Barnum (домен-проекция антипаттернов #4/#14). Профекция — не одна сфера: дом
плюс его управитель тянут 2–3, отсюда **рифмы года** (9-й/Венера связывает смысл и ценность).

**По-сферный чек-лист (полнота по домен-оси):** база(натал) → заряд(профекция/соляр) → сдвиг(прог/дир)
→ **узел напряжения** → **что на столе**. Подсвеченная сфера без клетки — дыра; тихая законно несёт
только базу плюс строку. Это **второй разрез** того же массива: реестр обхода режет по **факторам**,
чек-лист — по **областям**; оба про полноту, но домен-разрез — **бремя модели**, без машинерии.

**Членство фактора в сфере — диспозиция, не машинный фактор.** Связь many-to-many (Венера-домицил-8
несёт деньги · ценность · близость · общий ресурс разом), и назначение — толкование. Поэтому оно живёт
на стороне **модели** (как `coverage_dispositions`), а **не тегом** в `coverage_factors.csv`: тег
протащил бы суждение через шов скрипт↔модель (см. keyed-контракт). Полноту даёт чек-лист, не
переиндексация реестра.

**Разделение осей — по ОСИ ОРГАНИЗАЦИИ, не по наличию планет.** Различие окна и сферы НЕ в том, что в
окне планеты есть, а в сфере нет — оба насыщены астрологией. Различие в ОСИ: окно организовано по
ВРЕМЕНИ (хронология: «сентябрь — Уран на Меркурий, риск такой-то»); сфера организована по ТЕМЕ —
ПОСЛОЙНЫЙ срез области: натальная база (дома сферы + их управители + жильцы) → SR-оверлей (SR-планеты в
домах сферы) → транзиты/дирекции/прогрессии/склонения, что бьют в управителей сферы → итог. Сфера
ДОЛЖНА называть планеты, дома, аспекты и даты — детально, плотно. ОБЪЁМ СЛЕДУЕТ СОБЫТИЙНОСТИ, не числу:
богатая сфера может быть размером со ВСЕ окна вместе (500+ слов — норма, не предел), тихая — короткой
строкой; не надувать пустое, не резать богатое (образец детализации — реф-прогон «Ум/Дело/Любовь»,
и ГЛУБЖЕ). ⛔ ПРЕЖНЕЕ правило «сферы без планет/дат, убери все имена планет» — ОТМЕНЕНО (сессия
2026-06-19): оно душило сферы в обезличенную вату — ровно то, чем сфера НЕ должна быть. Анти-дубль (сфера ≠ пересказ окна) держится ОСЬЮ: сфера собирает факторы темы
ПОСЛОЙНО (вертикальный срез), а не повторяет хронологию окна. Анти-Barnum: тест НЕ «убери планеты», а
наоборот — «держится ли фраза на НЕСУЩЕЙ фактуре? если она работает на одном тёплом тоне без конкретных
планет/домов/аспектов — это вата, переписать». Конкретика ОБЯЗАТЕЛЬНА, а не изгнана.

**Планка подачи — названное давление плюс открытый исход.** Конкретный узел и что на столе,
вероятностно, без предрешения (продолжение недетерминизма и открытой валентности). Не вата «область
вообще» и не предсказание события. («Деньги — узел года: ценность проверяется под внешним требованием;
вопрос года — устоять в своей цене» — называет давление, не закрывает исход.)

**Число сфер — чек-лист, не квота.** Семь (редукция 12→немного) держим **за кулисами для полноты**,
чтобы ни одну область не забыть. **Подача идёт по карте:** вести подсвеченными, **рифмующиеся сферы
сливать в темы**, тихие — строкой. Сколько тем подано — следует карте, не сетке (на одной 3 темы, на
другой 5).

**Где живёт.** По-сферный разбор — в мастер-чтении (рабочая делинеация, с базисом-факторами); отчёт его
**сжимает** в домен-регистр (показывать вывод, не заверять; см. `prose-style-ru.md`).

## Проверка приёмки: воспроизводимо ли чтение из ОДНОГО дока
Берём уже сделанный вывод и собираем его **только из операторов + таблиц** этого дока. Где не
вышло — там дыра, молча заполненная из головы.

**Пример — «Венера в Тельце в 8-м, корень карты»:**
- ♀ (деятель: сближать/ценить; веер: связь↔вязкость) **⊗ Телец** (резервуар: телесное/устойчивое/
  медленное) → «ценить» в вещественно-устойчивом стиле = **ценность телесная, традиционная, медленная**.
- **⊗ 8-й дом** (арена: общий/чужой ресурс/раздел/глубина) → эта ценность проявляется в области
  **общего ресурса и его раздела**.
- **корень = диспозитор-сток** (оператор 3: большинство цепочек проведены к Венере) → этот деятель —
  **гравитационный центр**, вся карта разворачивается к нему.
- Композиция (веер, валентность открыта): **«телесно-традиционная ценность, проживаемая через
  общий ресурс / раздел / глубину»** — ровно прежний вывод. ✓ Воспроизведено из дока.
- Валентность (раздел vs обладание vs интимность) — **не предрешена**, выбирается наблюдением.

Критерий: новый вывод считается готовым, только если так же собирается из дока. Не собирается →
доработать таблицу/оператор, а не дописать смысл «из головы».

**Второй критерий приёмки — полнота.** Чтение готово, только если **реестр обхода закрыт**: у каждого
фактора непустая диспозиция, тихое помечено тихим (не выброшено), ранги стабилизировались. Пустая
клетка = молчаливый пропуск, чинить до сдачи. (См. «Полнота обхода».)

**Третий критерий приёмки — версии видимы** (gate 3): свёрнутый полюс показывает работу ветвей в логе
версий (`taken`/`parked`/`dropped`+основание); молчаливого схлопывания нет. (См. «Лог версий».) **Три
критерия независимы:** воспроизводимость ловит «дописал из головы», полнота — «пропустил/черипикнул
**фактор**», лог версий — «молча свернул **ветку** внутри фактора».
