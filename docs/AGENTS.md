# AGENTS.md

Local rules for `docs/`.

## Scope

- Applies to documentation files under `docs/`.

## Rules

1. Keep public-safe documentation in `docs/public/` when content can expose sensitive context.
2. Do not move private task/backlog/tasklog data into public docs.
3. When redacting examples, prefer neutral/public data and remove local personal identifiers.
4. If operational commands change, update `docs/RUNBOOK.md` and keep commands executable.
5. Keep docs concise for agents: index file + linked detailed files over large monolithic documents.
