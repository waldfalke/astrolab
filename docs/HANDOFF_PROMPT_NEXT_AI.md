# Задание для другого ИИ (нейтральная формулировка)

Нужно продолжить формализацию методологии фазового анализа (в духе подхода, обсужденного ранее), без мистификации и без навязывания интерпретаций.

## Цель

Собрать строгую и человекопонятную спецификацию вычисления состояния системы по:

1. фазе по знаку,
2. фазе по дому (Плацид),
3. диспозиции,
4. микро-фазам.

## Принятые допущения

1. Модель рабоче-вычислительно одномерная: цикл 360° (`S1`).
2. Знаки и дома читаются через единый фазовый словарь `1..12`.
3. Анализ должен быть формальным (формулы, шаги, нотация), а не повествовательным.

## Нотация (зафиксированная)

1. Код состояния планеты:
   - `P = (Z.z, H.h, D)`
   - `Z` — фаза по знаку от обители, `z` — микрофаза в знаке.
   - `H` — фаза по дому от ASC, `h` — микрофаза в доме.
   - `D` — фаза диспозитора.

2. Диспозиция:
   - `A -(k)-> B`, где `B` — диспозитор `A`, `k` — фазовый индекс `A`.

3. Аспектная связь (направленная):
   - верхний индекс: фазовый смысл,
   - нижний индекс: угол.
   - Не использовать симметричное `<->` как общую норму без отдельного формального обоснования.

## Формулы

1. Микрофаза в знаке:
   - `z = ceil(deg_in_sign / 2.5)`, диапазон `1..12`.

2. Микрофаза в доме (Плацид):
   - `L_house = cusp_next - cusp_current` (по кругу),
   - `L_pos = lon_planet - cusp_current` (по кругу),
   - `h = ceil(12 * L_pos / L_house)`, диапазон `1..12`.

## Что нужно выдать

1. Формальный документ-спецификацию:
   - термины,
   - формулы,
   - правила нотации,
   - порядок вычислений.
2. Справочник фаз `1..12` (один для знаков и домов).
3. Правила для графа:
   - построение,
   - подпись ребер,
   - поиск центров управления.
4. Раздел «границы модели»:
   - что входит в ядро,
   - что считается опциональным (например, склонения).
5. 3-5 тест-кейсов в формате «вход -> ожидаемый код».

## Ограничения стиля

1. Без манипулятивного тона.
2. Без утверждений «истина/ложь» вне формальной модели.
3. Ясно отделять:
   - доказуемые шаги вычисления,
   - допущения,
   - интерпретационные гипотезы.
