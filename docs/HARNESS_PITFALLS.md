# Harness pitfalls — recurring engineering traps

Short operational checklist. Read before editing recipes or running a client report. These are
classes we hit **more than once** — not one-off bugs. Reading-craft antipatterns (cherry-pick, Barnum)
live in `report-standards.md` / `semantic-base.md`; this file is the engineering layer.

| # | Symptom (how it shows) | Rule |
|---|---|---|
| 1 | **Computed ≠ surfaced.** A layer is calculated by a recipe but the model never uses it — because it isn't enumerated in coverage, OR not named in the BRIEF, OR not in the twin. Hit on phases, SR-declinations, directions, progressions, node-aspects, SR-nodes, transit-nodes. | A layer is "wired" only when ALL THREE meet: **compute** (orchestrator calls it) + **enumerate** (build_coverage_ledger lists the factor) + **name** (BRIEF tells the model). Miss one → silent drop (NKS astrolab #79). |
| 2 | **Decimal comma.** PowerShell writes `101,58` by locale; Python / the renderer can't parse it → empty fields or a crash. Hit on the validator, the transit-ring CSV, coverage parsing. | When a PS recipe writes numbers a Python step reads, force invariant culture (`.ToString([CultureInfo]::InvariantCulture)`) or use `Write-InvariantCsv`. Readers should also tolerate `replace(',','.')`. |
| 3 | **PII to a public path.** A recipe writes client data into the tracked `charts/` instead of `.private/` (phase_vectors did; sphere_ledger would have). Or client data gets attributed to the wrong person by guess. | Reproducible client work passes `-ChartsRoot $PrivateRoot`; the orchestrator guards `\.private\`. Never guess whose birth data a chart holds — verify from `chart.yaml` or ask. |
| 4 | **Silent data loss.** Input is truncated without a flag — birth seconds dropped by `%H:%M`; a layer quietly skipped. | Parse defensively and ERROR on the unexpected, never silently narrow. Seconds, timezones, optional fields — keep or fail loud. |
| 5 | **Contract / column-name drift.** Code reads a column that doesn't exist (`exact_date` vs `exact_dates`) → blank output that looks "done". Or prose.md section markers don't match the assembler. | When one recipe writes a CSV another reads, the column names ARE the contract — verify against the actual header, not memory. Same for the prose.md `[[SECTION]]` markers. |
| 6 | **Toothless gate.** A check flags a problem (HOLES, schema) but the pipeline proceeds anyway → the defect ships. The blind run assembled over 169 blank dispositions. | A gate that matters must `throw`, not warn. Block the deliverable; don't advise. Persist the gate output (Tee) so it's auditable. |
| 7 | **Script the emergent / hand-fake an artifact.** Reflex to determinize interpretation (kills the gestalt, NKS #73), or the model hand-draws a wheel / hand-builds a techblock instead of using the real computed one. | The LLM is the organ of the emergent READ — never script the reading. Conversely, numbers/wheels/techblocks are deterministic — the model writes PROSE only, never fabricates a computed artifact. |
| 8 | **Encoding crash.** `git diff` / subprocess output decoded as cp1251 chokes on UTF-8 (Cyrillic, ●◐⟨⟩) → hook or validator dies. | Read external process output as `encoding="utf-8", errors="replace"`; set `PYTHONIOENCODING=utf-8` for Python steps that print non-ASCII. |
| 9 | **Half-cwd-robust script.** A recipe works from the repo root but breaks after a `cd` into a subdir (relative script path, git run from the wrong cwd). | Resolve paths from the script's own location (`$PSScriptRoot` / `__file__`), and run `git` with an explicit `cwd=REPO`. |
| 10 | **House by eye, not by H_house.** The model reads a planet's house from proximity to an angle instead of the computed `H_house` — "Sun 3° under MC" written as "10th house" when it's the 9th. Caught by advisor before prose; leaks into the portrait + desyncs sections. | A planet's house ALWAYS comes from computed `H_house` (phase_vectors) / `sphere_routing` (Placidus), never eyeballed. Under-an-angle ≠ in that angular house. |
| 11 | **Magic number that should follow content.** Writing a fixed threshold (word count, orb, N) where the right size follows the material — e.g. "~150-250 words per sphere" when the team had decided spheres run 500+ and should GROW; the cap then strangles exactly what it was meant to enrich. | Size/threshold follows the content, not a number. Sphere length follows its eventfulness (rich → as long as all the windows combined; quiet → a line); don't inflate empty, don't cap rich. State the PRINCIPLE, never bake in a number a future case will exceed. |
| 12 | **Anxious imperative tone in the BRIEF.** Piling on CAPS + "MANDATORY / ONLY / else it's broken / ⚠️" out of fear of losing quality. Recurring: "нарушение=брак", advisor "don't overuse", the 150-250 cap, the phase "ОБЯЗАТЕЛЬНО". A frightened reader becomes an auditor, not a reader — kills the gestalt the whole harness exists to enable. | The BRIEF is a calm orientation, not a threat list. Give the frame, trust the reader, mark what's valuable or easy to miss — without "must/only/else-a-hole". Floor rules (don't fabricate, PII, gate) can be firm; craft guidance stays an invitation. |
| 13 | **Rule overfit to one chart.** Deriving a harness rule from what's loud on a single client — "drop Jupiter from SLOW_CARRIERS" because Jupiter bloated THIS chart's windows to 54. On the next chart it's Saturn or the nodes; cutting Jupiter loses real Jupiter-to-angle themes. Same class as the magic number (#11): "magic planet/chart". | Derive rules from MECHANICS (un-merged retro passes, perfection, weight), not from one chart's loud body. The real cause of window bloat is un-merged retro passes (planet-agnostic), not a specific planet. Verify any axis rule on ALL charts (owner + others + holdout), not the one you debugged on. |
| 14 | **Angles under-read (planet-centric reading).** Reading planets+aspects and letting the ANGLES (ASC-DSC, MC-IC) slip — yet a transit/SR/config to a natal angle outweighs the same to an ordinary planet (angularity = strength). Also conflating two axes: "approx time → angles ◐" is RELIABILITY; it must not bleed into "angles are secondary" (WEIGHT). At exact time, angle hits are firm ●. | Angles are a FIRST-CLASS layer — raise them into portrait and year alongside the lights and chart ruler. Distinguish transiting houses (spin over a day, no "house of the day") from natal angles (fixed, transits TO them are valid). Data exists (H_house, chart_points) — it's an attention gap, not a compute gap. Intermediate cusps + Vertex are a second priority; angles first (NKS #88). |

| 15 | **Layer whose key value is structurally unsampleable by the data window.** A new enum/feature whose defining case can't appear in the data the scan produces — zone classification with `tail` (slow transit perfecting just BEFORE the year) while the scan starts exactly AT the year anchor: `tail` never fires, the enum silently collapses to core/horizon, and a "test on N charts" PASSES by reading the absence as "these charts have no tail". A dead branch that looks correct. | Before trusting a classifier, check its data window can CONTAIN every class. Decouple the scan window from the semantic anchor (scan ±margin around the period; classify relative to the anchor). Test with a DISCRIMINATING case — one chart-year known to have the rare class — and confirm the value actually appears, never infer "feature works" from "no counterexample seen" (advisor caught this on zones #84). |

| 16 | **One recipe's two modes drift in what they emit.** A recipe with a range mode and a snapshot mode (or two providers behind them) silently emits a DIFFERENT target/field set in each. `run_transits_to_natal` range (swiss) carried natal angles ASC/MC/IC/DSC; snapshot (ephem, no chart_points) dropped them — so transit-day-via-snapshot under-read the angle layer (#88/#14) with no error. The provider difference (ephem has no angles) became a silent capability gap. | When a recipe has 2 modes/providers, assert they yield a COMPARABLE set (targets, columns, layers). A provider lacking a field (ephem→angles) is not a license to drop the layer — backfill from the other provider (swiss chart_points). Diff the two modes' outputs on the same input before trusting either. |
| 17 | **PowerShell `[int]` rounds (half-to-even), it does not truncate.** `[int]59.5 → 60`, so a fractional minute became the time string `HH:60`; same risk for indices, house numbers, degree buckets derived from a floating value. Looks like an off-by-one that "only happens sometimes". | For floor semantics (time fields, indices, bucket ids) use `[math]::Floor(...)`, never bare `[int]`. Reserve `[int]` for when you actually want rounding. Caught in the rising-hands watch table. |

When a NEW recurring trap shows up (hit twice), add a row here — symptom + rule, one line each. Keep it short.
