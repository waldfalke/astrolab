## Objective

Introduce strict schema validation for chart project contracts to prevent silent format drift.

## Problem Statement

1. `chart.yaml` and `INDEX.yaml` are generated by manual string assembly.
2. No schema validator enforces required fields and types.
3. YAML format errors or accidental field drift are currently undetected.

## Scope

1. Add schema definitions for:
   - `chart.yaml`
   - `INDEX.yaml`
2. Build validation command:
   - `validate_chart_project.ps1`
3. Validation checks:
   - schema compliance
   - required file existence
   - output-to-source mapping consistency

2. Runtime linting inside every recipe (phase 2).
