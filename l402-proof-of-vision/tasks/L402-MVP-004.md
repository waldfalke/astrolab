## Objective

Создать и упаковать в контейнер скрипт-агент, способный обработать L402-ответ и "оплатить" доступ к защищенному ресурсу.

## Problem Statement

1.  Для демонстрации end-to-end цикла L402 необходим клиент, который "понимает" этот протокол.
2.  Клиент должен уметь обработать ошибку `402`, извлечь данные для оплаты и совершить повторный запрос с авторизацией.
3.  Клиент должен быть контейнеризирован для работы в единой Docker-сети с `mock-api`.

## Scope

1.  Создание `package.json` с зависимостью `axios`.
2.  Создание `client.js`, реализующего "dummy" L402-клиентскую логику:
    - Первый запрос к `mock-api`.
    - Обработка ответа `402 Payment Required`.
    - Извлечение "dummy" токена из заголовка `WWW-Authenticate`.
    - Формирование `Authorization` заголовка с "dummy" токеном и "dummy" preimage.
    - Повторный запрос с авторизацией.
    - Вывод финального результата в консоль.
3.  Создание `Dockerfile` для запуска скрипта.
4.  Добавление сервиса `client-agent` в `docker-compose.yml`.

## Out of Scope

1.  Реальная оплата в криптовалюте. Вся логика оплаты имитируется.
2.  Сложная логика парсинга или обработки ошибок.

## Done Definition

1.  Все файлы (`package.json`, `client.js`, `Dockerfile`) созданы в `client-agent/`.
2.  Сервис `client-agent` определен в `docker-compose.yml`.
3.  Команда `docker compose build client-agent` успешно завершается.

## Implementation

1.  В директории `client-agent` созданы `package.json` (с `axios`), `client.js` и `Dockerfile`.
2.  В `client.js` написана асинхронная функция `getPaidResource`, которая:
    - Делает первый запрос, ожидания ошибку 402.
    - В блоке `catch` обрабатывает ошибку, извлекает токен из заголовка ответа.
    - Формирует заголовок `Authorization: L402 <token>:<preimage>` с использованием захардкоженного `DUMMY_PREIMAGE`.
    - Делает второй, успешный запрос с этим заголовком.
    - Выводит полученные данные в консоль.
3.  В `docker-compose.yml` добавлен сервис `client-agent`, который собирается из `client-agent/` и зависит от `mock-api`.
