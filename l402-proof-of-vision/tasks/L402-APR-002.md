# [TODO][TASK][P0] L402-APR-002 — Add lnd-client and setup-channel script

## Goal
Добавить второй LND-узел (lnd-client) для клиентской стороны и скрипт для создания funded канала между двумя узлами.

## Problem Statement
Один LND не может оплатить свой собственный инвойс. Для реального Lightning-платежа нужны два узла с открытым каналом.

## Scope
1. **lnd-client в docker-compose.yml** (уже частично сделано):
   - Отдельный volume `lnd_client_data`
   - `--tlsextradomain=lnd-client`
   - healthcheck аналогичный lnd
2. **`scripts/setup-channel.sh`:**
   - `bitcoin-cli createwallet "default"` (если не создан)
   - Mine 101 блоков для maturity
   - Получить адрес от lnd-client (`lncli --network=regtest newaddress p2wkh`)
   - Отправить BTC с bitcoind на lnd-client
   - Mine 6 блоков для подтверждения
   - Подключить lnd-client к lnd (`lncli connect <pubkey>@lnd:9735`)
   - Открыть канал (`lncli openchannel --node_key=<pubkey> --local_amt=500000`)
   - Mine 6 блоков для подтверждения канала
   - Verify: `lncli listchannels` показывает активный канал

## Depends on
- Рабочий bitcoind + lnd (Phase 1)

## Done Definition
1. `docker compose up -d bitcoind lnd lnd-client` — все три healthy.
2. `./scripts/setup-channel.sh` завершается без ошибок.
3. `lncli --network=regtest listchannels` на lnd-client показывает канал с балансом.
