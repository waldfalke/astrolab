# [DONE][TASK][P1] L402-MVP-006 - Integrate Real Lightning Node (LND)

## Goal
Заменить "dummy" L402-логику в `mock-api` на интеграцию с настоящим узлом Lightning Network (LND) для генерации и проверки реальных счетов.

## Problem Statement
1.  Текущая реализация MVP использует "dummy" логику и не доказывает возможность взаимодействия с реальной крипто-системой.
2.  Для полноценного "пруфа" необходимо продемонстрировать, что наш сервис может программно создавать и проверять крипто-счета.

## Scope
1.  **Добавление `bitcoind` в `docker-compose.yml`:**
    - Использовать готовый образ.
    - Настроить для работы в режиме `regtest` (локальная тестовая сеть).
2.  **Добавление `lnd` в `docker-compose.yml`:**
    - Использовать официальный образ `lightninglabs/lnd`.
    - Настроить для работы в `regtest`, связать с `bitcoind`.
    - Пробросить порты для gRPC и REST.
3.  **Модификация `mock-api`:**
    - Добавить gRPC-клиент и зависимости для работы с `lnd` в `package.json`.
    - Изменить `server.js`: при запросе без авторизации он должен делать gRPC-вызов к `lnd` для создания нового счета (`addInvoice`).
    - Возвращать в заголовке `WWW-Authenticate` настоящий счет от `lnd`.
    - При запросе с авторизацией, `server.js` должен делать gRPC-вызов к `lnd` для проверки оплаты счета (`lookupInvoice`).

## Out of Scope
1.  Настройка макросов или других сложных функций `lnd`.
2.  Настройка и тестирование `client-agent` для работы с реальным `lnd` (это будет следующая задача).
3.  Развертывание в основной сети (`mainnet`). Вся работа ведется в `regtest`.

## Done Definition
1.  `docker-compose.yml` содержит рабочие конфигурации для `bitcoind` и `lnd`.
2.  Команда `docker compose up` успешно запускает все три сервиса (`bitcoind`, `lnd`, `mock-api`).
3.  `mock-api` при запуске успешно соединяется с gRPC-интерфейсом `lnd`.
4.  Запрос `curl -i http://localhost:8080/sun_sign` возвращает HTTP-статус `402` и заголовок `WWW-Authenticate`, содержащий настоящий `regtest`-счет, сгенерированный `lnd`.
