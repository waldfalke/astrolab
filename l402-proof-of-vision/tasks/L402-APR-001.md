## Goal
Добавить Aperture в docker-compose как L402 reverse proxy перед mock-api.

## Problem Statement
Сейчас mock-api сам генерирует инвойсы через gRPC к LND. Эту логику нужно вынести в Aperture — production-ready L402 прокси от Lightning Labs.

## Scope
1. Добавить сервис `aperture` в `docker-compose.yml`:
   - Image: `lightninglabs/aperture:v0.3-beta`
   - depends_on: `lnd: condition: service_healthy`
   - Volume: `lnd_data:/lnd:ro` (для tls.cert и macaroons)
   - Port: `8081:8081` (внешний L402 endpoint)
2. Создать `aperture.yaml`:
   - `authenticator.network: "regtest"`
   - `authenticator.lndhost: "lnd:10009"`
   - TLS cert и macaroon paths внутри контейнера
   - `insecure: true` (для regtest — без Let's Encrypt)
   - Service: `mock-api:3000`, price: 1000 sats
3. Проверить что Aperture стартует и подключается к LND.

## Depends on
- Рабочий `bitcoind` + `lnd` (есть из Phase 1)

## Done Definition
1. `docker compose up -d aperture` — контейнер стартует без ошибок.
2. В логах Aperture видно подключение к LND.
